Standard Title Search

By default, when you enter text into the search field, it will do a case insensitive, contains, title search. For example, if you enter big bang it will find titles that contain big bang such as The Big Bang Theory.

PQL Searches

You can also use PQL (Phoenix Query Language) to construct simple or complex searches against metadata fields. To use PQL querries, simply prefix your query with pql

Examples

pql Genre contains 'Horror' or Genre contains 'Thriller'
Will search for all media files where the genre contains horror or thriller
pql MediaType is null
Will find media files where the MediaType has not been set
You can also group conditions and use and or or to create a more complex query. The following query is basically what BMT uses to find missing metadata
MediaType is null or MediaTitle is null or (MediaType = 'TV' and EpisodeNumber = 0)

Notes

  • The field names are case sensitive (ie, Title, MediaTitle, etc)
  • and and or is case sensitive
  • The following operands are supported: =, >, <, <>, is null, is not null, contains
  • The complete list of Metadata is not nicely documented but you can decipher the Field Names from this file.
